# 故障排除指南

本指南帮助您解决交易过程中遇到的各种技术问题和故障。请按照以下步骤进行问题排查，如果问题仍然存在，请联系技术支持。

## 🚨 紧急问题快速通道

### 立即需要帮助？
如果遇到以下紧急情况，请优先处理：

1. **账户被锁定** → 联系24/7客服
2. **资金异常减少** → 立即修改密码并联系安全团队
3. **无法登录交易** → 检查网络和平台状态
4. **重要订单无法执行** → 使用备用平台或联系交易台

## 🔌 连接问题

### 无法连接到服务器

#### 症状
- 平台显示"无连接"或"离线"状态
- 登录时提示"无效账户"或"服务器不可用"
- 价格数据不更新

#### 排查步骤

**第一步：检查网络连接**
```bash
# Windows命令提示符
ping server.company.com
tracert server.company.com

# Mac终端
ping server.company.com
traceroute server.company.com
```

**第二步：验证服务器状态**
1. 访问官网"服务器状态"页面
2. 检查是否有维护公告
3. 查看社交媒体或邮件通知

**第三步：平台设置检查**
1. **MT4/MT5**: 工具 → 选项 → 服务器 → 选择正确的服务器
2. **Web平台**: 清除浏览器缓存，使用无痕模式
3. **移动端**: 检查WiFi/移动数据切换

**第四步：网络环境调整**
- 尝试切换网络（WiFi/移动热点/有线）
- 暂时禁用防火墙和杀毒软件
- 使用VPN连接（如受地域限制）

### 延迟和断线问题

#### 常见原因
- **网络拥塞**: 高峰时段网络质量下降
- **服务器负载**: 交易活跃时段服务器压力大
- **本地问题**: 路由器、ISP或设备问题

#### 优化建议
1. **有线连接**: 优先使用网线而非WiFi
2. **关闭占用带宽的应用**: 视频流、大文件下载
3. **选择最近服务器**: 选择物理位置最近的服务器
4. **优化平台设置**: 减少同时打开的图表数量

## 💻 平台技术问题

### MT4/MT5常见问题

#### 1. 平台启动失败
**症状**: 双击无反应、闪退、启动时卡住

**解决方案**:
1. **以管理员身份运行**
   - 右键图标 → 以管理员身份运行
2. **兼容性模式**
   - 右键图标 → 属性 → 兼容性 → 以兼容模式运行
3. **清理缓存**
   - 删除`C:\Users\[用户名]\AppData\Roaming\MetaQuotes`下的缓存文件
4. **重新安装**
   - 完全卸载后重新下载安装

#### 2. 图表显示异常
**症状**: 图表空白、K线不显示、指标异常

**解决方案**:
1. **重置图表**
   - 右键图表 → 模板 → 默认
2. **重新下载数据**
   - 工具 → 历史数据中心 → 下载
3. **检查时间设置**
   - 工具 → 选项 → 服务器 → 启用DST
4. **清理历史数据**
   - 工具 → 选项 → 图表 → 历史数据最大K线数

#### 3. EA/指标不工作
**症状**: EA不执行交易、指标不显示、脚本错误

**解决方案**:
1. **检查自动交易开关**
   - 确保工具栏"自动交易"按钮为绿色
2. **查看日志**
   - 终端 → 日志 → 查看错误信息
3. **重新编译**
   - 打开.mq4文件 → 编译（F7）
4. **权限检查**
   - 右键EA → 属性 → 允许实时交易

#### 4. 策略测试器问题
**症状**: 回测结果不准确、测试速度慢、数据缺失

**解决方案**:
1. **数据质量**
   - 使用1分钟M1数据回测最准确
   - 检查数据完整性
2. **测试设置**
   - 使用"每个即时价位"模式
   - 开启"优化"模式进行参数测试
3. **内存管理**
   - 关闭其他程序释放内存
   - 减少测试的品种和时间范围

### Web平台问题

#### 浏览器兼容性问题
**支持浏览器**:
- ✅ Chrome 80+（推荐）
- ✅ Firefox 75+
- ✅ Safari 13+
- ✅ Edge 80+
- ❌ Internet Explorer（不再支持）

#### 常见Web问题
1. **页面加载失败**
   - 清除浏览器缓存（Ctrl+Shift+Delete）
   - 禁用浏览器插件逐一排查
   - 尝试无痕/隐私模式

2. **交易按钮无响应**
   - 检查JavaScript是否启用
   - 更新浏览器到最新版本
   - 禁用广告拦截器

3. **图表显示问题**
   - 检查WebGL支持（chrome://gpu）
   - 更新显卡驱动程序
   - 降低图表复杂度

### 移动端问题

#### 应用崩溃或闪退
1. **更新应用**: 应用商店下载最新版本
2. **清理缓存**: 设置 → 应用 → 清除缓存
3. **重启设备**: 完全关闭后重新启动
4. **重新安装**: 卸载后重新安装

#### 通知不工作
1. **系统权限**: 检查通知权限是否开启
2. **电池优化**: 关闭应用电池优化
3. **后台运行**: 允许应用在后台运行
4. **通知设置**: 应用内开启相关通知

## 💰 交易相关问题

### 订单执行问题

#### 订单被拒绝
**常见原因和解决方案**:

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| **无效价格** | 价格变动过快 | 使用市价单，放宽价格容忍度 |
| **资金不足** | 保证金不够 | 入金或减少交易量 |
| **超出限制** | 违反交易规则 | 检查账户限制，调整订单参数 |
| **市场关闭** | 非交易时间 | 查看交易时间表，等待市场开盘 |
| **无效手数** | 交易量不符合要求 | 调整到最小/最大手数范围内 |

#### 订单延迟执行
1. **网络延迟**: 使用ping测试网络延迟
2. **服务器负载**: 避开高峰交易时段
3. **平台设置**: 调整执行模式（即时执行/请求执行）
4. **订单类型**: 市价单比挂单执行更快

### 资金和账户问题

#### 存款/取款延迟
**标准处理时间**:
- **电子支付**: 即时-24小时
- **银行转账**: 1-3个工作日
- **信用卡**: 即时-几小时

**延迟排查**:
1. **检查邮件**: 确认收到处理确认邮件
2. **联系客服**: 提供交易参考号查询状态
3. **银行确认**: 联系银行确认转账状态
4. **文件验证**: 确保所有验证文件已提交

#### 账户余额异常
1. **刷新平台**: 平台可能显示缓存数据
2. **查看历史**: 账户历史中查看所有资金变动
3. **核对订单**: 检查是否有未平仓订单影响
4. **联系财务**: 如有差异立即联系财务部门

## 📊 数据和报价问题

### 价格数据不准确

#### 数据差异原因
1. **数据源不同**: 不同平台使用不同数据源
2. **延迟差异**: 免费数据和实时数据有延迟
3. **报价方式**: 买入价/卖出价显示差异
4. **时间同步**: 平台时间设置不同

#### 解决方案
1. **统一数据源**: 使用同一数据源进行对比
2. **启用实时数据**: 购买或订阅实时数据
3. **时间校准**: 同步平台和系统时间
4. **联系技术支持**: 报告持续的数据问题

### 历史数据缺失

#### MT4/MT5数据恢复
1. **自动下载**: 工具 → 选项 → 图表 → 最大柱数
2. **手动下载**: 工具 → 历史数据中心 → 下载
3. **数据导入**: 从第三方来源导入数据
4. **更换服务器**: 不同服务器可能有不同历史数据

## 🛠️ 高级故障排除

### 系统日志分析

#### MT4/MT5日志位置
```
C:\Users\[用户名]\AppData\Roaming\MetaQuotes\Terminal\[终端ID]\logs\
```

#### 重要日志文件
- **logfile.log**: 主要操作日志
- **tester.log**: 策略测试器日志
- **experts.log**: EA执行日志
- **journal.log**: 平台运行日志

### 网络诊断工具

#### Windows网络诊断
```bash
# 基本连接测试
ping 8.8.8.8
ping server.company.com

# 路由跟踪
tracert server.company.com

# 网络状态
netstat -an | findstr "ESTABLISHED"

# 重置网络
ipconfig /flushdns
netsh winsock reset
```

#### Mac网络诊断
```bash
# 基本连接测试
ping 8.8.8.8
ping server.company.com

# 路由跟踪
traceroute server.company.com

# DNS缓存清理
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```
